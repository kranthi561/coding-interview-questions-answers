# 3Sum

---

1. What is meant by 3Sum?

## What is 3Sum?

Given an integer array `nums`, return all **unique triplets** `[nums[i], nums[j], nums[k]]` such that `i != j != k != i` and `nums[i] + nums[j] + nums[k] == 0`.

**The solution must not contain duplicate triplets.**

**Example:**
```
Input:  nums = [-1, 0, 1, 2, -1, -4]
Output: [[-1, -1, 2], [-1, 0, 1]]
```

```
Input:  nums = [0, 1, 1]
Output: []
```

```
Input:  nums = [0, 0, 0]
Output: [[0, 0, 0]]
```

2. Clarify Requirements

## Requirements

### Functional Requirements
- Find all unique triplets that sum to zero.
- No duplicate triplets in output.
- Order within each triplet and between triplets does not matter.

### Non-Functional Requirements
| Concern | Expectation |
|---------|-------------|
| **Scale** | Up to 3000 elements |
| **Duplicates** | Input may have duplicates; output must not |
| **Time** | O(n²) expected |

### Clarifying Questions
- Can the same element be used multiple times? → No, different indices.
- Is output order required? → No.
- Can input have duplicates? → Yes — must deduplicate output.

3. Time and Space Complexity

## Complexity Analysis

| Approach | Time | Space |
|----------|------|-------|
| Brute force (3 loops) | O(n³) | O(1) |
| Hash set for third | O(n²) | O(n) |
| **Sort + Two Pointers** | **O(n²)** | **O(1)** extra |

**O(n²)** is optimal for this problem — we need to enumerate all valid triplets.


---

4. Algorithm Choice and Why

## Algorithm: Sort + Two Pointers

### Steps
1. **Sort** the array: O(n log n)
2. For each element at index `i` as the **first element** of the triplet:
   - Use **two pointers** (`left = i+1`, `right = n-1`) to find the pair that sums to `-nums[i]`
   - If sum < 0 → move `left` right
   - If sum > 0 → move `right` left
   - If sum == 0 → record triplet, skip duplicates
3. Skip duplicate values for `i`, `left`, and `right` to avoid duplicate triplets.

### Why Sort First?
- Sorting enables two pointers (linear pair search).
- Sorting also makes deduplication easy — skip consecutive equal values.

5. High-Level Design

## High-Level Design

### Data Flow
```
nums = [-1, 0, 1, 2, -1, -4]
Sorted: [-4, -1, -1, 0, 1, 2]

i=0, nums[i]=-4, target=4
  left=1, right=5: -1+2=1 < 4 → left++
  left=2, right=5: -1+2=1 < 4 → left++
  left=3, right=5: 0+2=2 < 4 → left++
  left=4, right=5: 1+2=3 < 4 → left++
  left==right → skip

i=1, nums[i]=-1, target=1
  left=2, right=5: -1+2=1 == 1 → add [-1,-1,2], skip dupes
  left=3, right=4: 0+1=1 == 1 → add [-1,0,1], skip dupes
  left==right → skip

i=2, nums[i]=-1 (same as i=1) → SKIP DUPLICATE

i=3, nums[i]=0, target=0
  left=4, right=5: 1+2=3 > 0 → right--
  left==right → skip

Result: [[-1,-1,2],[-1,0,1]]
```

6. Java Solution with Explanation

## Java Implementation

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;

public class ThreeSum {

    /**
     * Finds all unique triplets in nums that sum to zero.
     * Sort first, then use two-pointer technique for each fixed element.
     * Skip duplicates at each pointer to avoid duplicate triplets.
     *
     * @param nums input integer array
     * @return list of all unique triplets summing to 0
     */
    public List<List<Integer>> threeSum(int[] nums) {
        List<List<Integer>> result = new ArrayList<>();
        Arrays.sort(nums);

        for (int i = 0; i < nums.length - 2; i++) {
            // Skip duplicate values for the first element
            if (i > 0 && nums[i] == nums[i - 1]) continue;

            // Early exit: smallest possible sum is already > 0
            if (nums[i] > 0) break;

            int left = i + 1;
            int right = nums.length - 1;

            while (left < right) {
                int sum = nums[i] + nums[left] + nums[right];

                if (sum == 0) {
                    result.add(Arrays.asList(nums[i], nums[left], nums[right]));
                    // Skip duplicates for left and right
                    while (left < right && nums[left] == nums[left + 1]) left++;
                    while (left < right && nums[right] == nums[right - 1]) right--;
                    left++;
                    right--;
                } else if (sum < 0) {
                    left++;
                } else {
                    right--;
                }
            }
        }

        return result;
    }

    public static void main(String[] args) {
        ThreeSum solution = new ThreeSum();

        System.out.println(solution.threeSum(new int[]{-1, 0, 1, 2, -1, -4}));
        // [[-1, -1, 2], [-1, 0, 1]]

        System.out.println(solution.threeSum(new int[]{0, 1, 1}));
        // []

        System.out.println(solution.threeSum(new int[]{0, 0, 0}));
        // [[0, 0, 0]]
    }
}
```

### Output
```
[[-1, -1, 2], [-1, 0, 1]]
[]
[[0, 0, 0]]
```

7. Python Solution with Explanation

## Python Implementation

```python
def three_sum(nums: list[int]) -> list[list[int]]:
    """
    Find all unique triplets that sum to zero.

    Strategy: Sort + Two Pointers.
    Fix first element, use two pointers for the remaining pair.
    Skip duplicates to avoid duplicate triplets in output.

    Time:  O(n²)
    Space: O(1) extra (output not counted)
    """
    nums.sort()
    result = []

    for i in range(len(nums) - 2):
        # Skip duplicate first element
        if i > 0 and nums[i] == nums[i - 1]:
            continue

        # All remaining sums will be positive — stop early
        if nums[i] > 0:
            break

        left, right = i + 1, len(nums) - 1

        while left < right:
            total = nums[i] + nums[left] + nums[right]

            if total == 0:
                result.append([nums[i], nums[left], nums[right]])
                # Skip duplicates
                while left < right and nums[left] == nums[left + 1]:
                    left += 1
                while left < right and nums[right] == nums[right - 1]:
                    right -= 1
                left += 1
                right -= 1
            elif total < 0:
                left += 1
            else:
                right -= 1

    return result


if __name__ == "__main__":
    print(three_sum([-1, 0, 1, 2, -1, -4]))  # [[-1,-1,2],[-1,0,1]]
    print(three_sum([0, 1, 1]))               # []
    print(three_sum([0, 0, 0]))              # [[0,0,0]]
```

### Output
```
[[-1, -1, 2], [-1, 0, 1]]
[]
[[0, 0, 0]]
```

8. Interview Q&A

## Q&A

**Q: Why sort the array first?**  
A: Sorting enables two-pointer technique for O(n) pair search per fixed element. It also makes deduplication easy — skip consecutive equal values.

**Q: How are duplicate triplets avoided?**  
A: Three places to skip duplicates:
1. Outer loop: `if i > 0 && nums[i] == nums[i-1] → continue`
2. Left pointer: advance past equal values after recording
3. Right pointer: retreat past equal values after recording

**Q: Why break early when `nums[i] > 0`?**  
A: Array is sorted. If the smallest element in the remaining triplet is positive, the sum can never be 0.

**Q: Can this be solved with a hash set instead of two pointers?**  
A: Yes — fix two elements, look up the third in a set. O(n²) time but O(n) space. Two pointers are cleaner and O(1) space.

**Q: What is the time complexity exactly?**  
A: O(n log n) for sorting + O(n²) for the two-pointer loops = O(n²) overall.

9. Summary

## Summary

| Attribute | Value |
|-----------|-------|
| **Problem** | Find all unique triplets summing to zero |
| **Optimal Algorithm** | Sort + Two Pointers |
| **Time Complexity** | O(n²) |
| **Space Complexity** | O(1) extra |
| **Key Insight** | Fix one element; find the complementary pair with two pointers; skip duplicates |

### Core Pattern
> Reduce k-sum to (k-1)-sum by fixing one element and recursing/looping. Two pointers solve the 2-sum subproblem in O(n) on a sorted array.
